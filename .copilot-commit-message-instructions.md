# コミットメッセージ生成規定

新規追加・修正したファイルに関するコミットメッセージは、以下の規定に従って作成してください。

* コミットメッセージのタイトルは英語で書き、50文字以内に収めること。
* タイトルの後に空行を入れ、その後に詳細な説明を記載することができます。詳細な説明は72文字以内で改行してください。
* その後、空行を入れ、変更概要を箇条書きリストで示してください。
* タイトルには、変更内容を簡潔に表す動詞+": " を使用してください（例: "Add: ", "Fix: ", "Update: ", "Remove: " など）。
* 関連するファイル名や関数名を含めると、変更内容が明確になります。
* コミットメッセージは、変更の理由や背景を説明するために使用されることがあります。
