## 1. 連続側の「中心バランス点」は何を最小化/停留させるか

ぬしの Big/Gap/Body を連続化するなら、自然なのは

\[
z^n-y^n=x^n
\quad\Longleftrightarrow\quad
n\log z=\log\!\bigl(e^{n\log y}+e^{n\log x}\bigr)
\]

ここで

\[
X:=\log x,\ Y:=\log y,\ Z:=\log z
\]

と置くと

\[
Z=\frac1n\log\!\bigl(e^{nX}+e^{nY}\bigr)
\]

右辺は **softmax（滑らかな max）** の形じゃ。
\(n\) が大きいほど \(\max(X,Y)\) に寄るし、\(n\to 0^+\) では平均に寄る。

この式が見せる「中心バランス点」はだいたい：

- \(n\) が小さい → 和の“均し”が強く、バランス（中庸）が作れる
- \(n\) が大きい → 片方が支配し、中心は崩れる

ぬしの「\(x^n\to e^k\)」は、まさに \(k=n\log x\) で、この座標変換そのものじゃ。

## 3. ぬしが欲しい「境界」を、ちゃんと数学的対象として立てる

「境界」を一言で言うなら：

\[
\textbf{連続最適（中心）} \quad\text{vs}\quad \textbf{整数格子の整合条件}
\]

これを具体化するなら、たとえばこんな関数が自然じゃ：

### (A) 連続の誤差関数

\[
E(x,y,z;n):=\log(z^n-y^n)-n\log x
\]

連続では \(E=0\) にできる（\(x=(z^n-y^n)^{1/n}\)）。

### (B) 整数スナップ誤差（最近接整数）

\[
x_0:=\left\lfloor (z^n-y^n)^{1/n} \right\rceil,\qquad
\Delta:=z^n-y^n-x_0^n
\]

ここで「境界」は、\(\Delta=0\) が起こるかどうか、つまり完全一致が可能か。

そしてこの \(\Delta\) を割る素数 \(q\) の付値が

\[
v_q(z^n-y^n)=v_q(x_0^n)=n\,v_q(x_0)
\]

の形に揃わないなら、\(\Delta=0\) は絶対に起こらぬ。

つまり、**連続→整数**へ落とすとき、境界は

\[
v_q(z^n-y^n)\equiv 0\pmod n
\]

という合同条件（段差条件）として現れる。
ここでまた `padicValNat` が出る。これは「整数解析しかしてない」のではなく、**整数へ落とす瞬間に不可避な条件**なのじゃ。

## 2. 「新しい素数が必ず出る」を本当に言うには、対象を \(S\) だけでなく“列”にする

単発の \(S\to S+1\) では「新しい素数の出現」を保証できぬ。
しかし、ぬしが欲しいのは単発ではなく

> 単位核 → 花弁展開 → 単位核完成 → …（反復）

という **反復生成系**じゃな？

このとき“新しい素数が必ず出る”に最も近い古典の定式化は **ユークリッドの証明**じゃ：

\[
P:=\prod_{p\in \mathcal P}p,\quad P+1
\]

有限集合 \(\mathcal P\) の素数全部を掛けた \(P\) に対して、
\(P+1\) はそのどれでも割れない。だから \(P+1\) には

- \(\mathcal P\) に含まれない素数因子が必ず存在する

これが「新しい素数が生まれる」の**堅い定理版**じゃ。

ぬしの言葉に寄せれば：

- **単位核 \(P\)**（既知素数の核）
- **+1 が成長1周期**
- **花弁 \(P+1\)**（既知核と互いに素の因子群）
- その花弁からまた核を更新して繰り返す

まさに「単位核→花弁展開→単位核→…」が、ユークリッド構造として既に完成しておる。

## 1. ぬしの言う「棒の構造」とは何か（数学的に固定する）

ユークリッド互除法は、数を **商と余りの列**に落とす：

\[
a = q_0 b + r_0,\quad
b = q_1 r_0 + r_1,\quad
r_0 = q_2 r_1 + r_2,\ \dots
\]

この列の「棒」感は、実は **連分数**や **格子上の最短ベクトル**の像じゃが、ぬしの世界では

- 単位核（unit）
- 2セットで1ユニット（白銀単位解釈）
- その繰り返しで尺度が組み上がる

という “単位のペアリング” が主役になる。

ここで「新しい gcd 定理」になりやすい核は、だいたい次の型じゃ：

### (A) gcd は「尺度の最小生成子」である（尺度原理）

\[
\gcd(a,b)=\min\{d\in\mathbb N\mid d\mid a \land d\mid b \}
\]

…は古典じゃが、ぬしの言う「尺度原理」はむしろ

> “棒”の1ステップが **尺度の更新**になっており、その不変量が gcd

という形になりやすい。

### (B) gcd の不変量：差分・線形結合・ステップ更新

互除法の一歩は

\[
(a,b)\mapsto (b, a\bmod b)
\]

これに対する不変量が

\[
\gcd(a,b)=\gcd(b,a\bmod b)
\]

で、ここに「棒の解析」が乗る。

## 2. ぬしの「白銀単位：2セット1ユニット」で gcd を組むとは

1. **偶奇（2進）で層を分ける**：

   v_2(a), v_2(b) \ \text{を剥がしてから本体の gcd を見る}
   \]

2. **\((a,b)\) を 2成分ベクトル（単位ペア）として扱う**：

## 3. ここで「追加されそうな gcd 定理」の候補を、3本に絞る

ぬしの文脈（素数尺度原理＋白銀2セット）から、追加されやすいのはこのへんじゃ：

### 候補1：2進剥離＋gcd の分解（binary gcd 的定理）

\[
\gcd(a,b)=2^{\min(v_2(a),v_2(b))}\cdot \gcd(a',b')

ここで \(a',b'\) は 2で割れない（奇数化）したもの。

これを「2セット1ユニット」の数学的骨にできる。

### 候補2：互除法ステップの“尺度”単調性

\[
r_{i+1} < r_i,\quad \gcd(r_i,r_{i+1})=\gcd(a,b)
\]

棒が短くなりつつ、不変量が維持される、という「構造解析」そのもの。

### 候補3：gcd が素数尺度の“最初の測定値”になる定理

素数 \(p\) ごとに

\[
v_p(\gcd(a,b))=\min(v_p(a),v_p(b))
\]

## 1. “半位相”をどう定義するか：最短は 2進付値での正規化

偶奇＝最下位ビットは、\(v_2\)（2進付値）で完全に表現できる。

\[
v_2(n)=\max\{k\in\mathbb N\mid 2^k\mid n\}
\]

そして「偶数は半位相落とす」は、

\[
\operatorname{odd}(n):=\frac{n}{2^{v_2(n)}}
\]

（\(n\) の 2冪因子を全部落として奇数化）
これが、ぬしの言う “0.5 の位相” を **整数だけで表す**最も安全な方法じゃ。

- “\(n\) を 1/2 ずらす” を **実数でやらず**
- “\(n\) から 2 の因子を剥ぐ” で表す

ここが Lean でも形式化しやすい。

## 2. ぬしの観察を定理に直す

### (A) 奇数 \(a\) について：\(\gcd(a,a+1)=1\) かつ \(\gcd(a,a+2)=1\)

奇数 \(a\) は \(a\) と \(a+1\) が互いに素なのは当然で、さらに

\[
\gcd(a,a+2)=\gcd(a,2)
\]

だから \(a\) が奇数なら \(\gcd(a,a+2)=1\)。
ぬしの例

- \(\gcd(1,2)=1 \Leftrightarrow \gcd(1,3)=1\)
- \(\gcd(5,6)=1 \Leftrightarrow \gcd(5,7)=1\)

はこの「+1 と +2 の互換が奇数で成り立つ」現象を見ておる。

### (B) 偶数 \(n\) の場合：\(\gcd(n,n+2)\) は必ず 2 以上

偶数 \(n=2m\) なら

\[
\gcd(n,n+2)=\gcd(2m,2m+2)=2\gcd(m,m+1)=2
\]

よって **必ず 2**。
だから

\[
\gcd(10,12)=2
\]

は必然じゃ。

そしてここがぬしの「半位相落とし」の核心：

\[
\gcd(2m,2m+2)=2 \quad\Longleftrightarrow\quad \gcd(m,m+1)=1
\]

つまり

\[
\gcd(10,12)=2 \Longleftrightarrow \gcd(5,6)=1
\]

であり、ぬしが書いた \(\gcd(5,7)=1\) は「+2 側」を見ているバージョンじゃな。
（奇数同士なら \(\gcd(5,7)=1\) は確かに成り立つが、互除法の対応として自然なのは \(m,m+1\) の方。）

## 3. 「0b11110000.0 / .1」という表現の正体

これは結局、

- 整数 \(N\) を「整数部」として固定し
- **半単位の位相（LSB の 0/1）**を付けて数える

という “2-adic 的アドレス化” に近い。

ただし、実数の「0.0/0.1」を使うと gcd の定義域が崩れるので、数学としては

\[
\text{半位相つき数} := (N,\epsilon)\quad(\epsilon\in\{0,1\})
\]

みたいに **ペア**で持つのが正しい。
白銀単位「2セット=1ユニット」は、この \(\epsilon\) のトグルとして解釈できる。

## 4. 白銀単位観点での“意味づけ”を一行で言うなら

白銀単位の「2セット」を、整数側ではこう読む：

> **2進尺度 \(v_2\) を剥いで odd 部分で比較する**のが、位相を揃えた gcd。

定義としては

\[
\operatorname{gcd_phase}(a,b):=\gcd(\operatorname{odd}(a),\operatorname{odd}(b))
\]

あるいはスケールも残すなら

\[
\gcd(a,b)=2^{\min(v_2(a),v_2(b))}\cdot \gcd(\operatorname{odd}(a),\operatorname{odd}(b))
\]

これが「偶数は半位相落とす必要がある」を、嘘なく数学に落とした形じゃ。

  \(\operatorname{odd}\) 正規化か、\((N,\epsilon)\) のペア構造で表すのが筋。

## 2. 「ペア整数（位相付き整数）」で 6 と 7 を同一視する最小モデル

いちばん筋の良い定式化は、相手 \(b\) を

\[
b = 2k + \varepsilon,\quad \varepsilon\in\{0,1\}
\]

と分解して **ペア \((k,\varepsilon)\)** として扱うことじゃ。

- \(6 = 2\cdot 3 + 0 \Rightarrow (3,0)\)
- \(7 = 2\cdot 3 + 1 \Rightarrow (3,1)\)

ここで「5 から見た相手は 6=7」というのを数学にするなら、

> 「5 を奇数として固定したとき、\(\varepsilon\)（LSB位相）を潰して \(k\) だけを見る射影を取る」

という操作になる。

射影：

\[
\pi(b)=\left\lfloor \frac{b}{2}\right\rfloor = k
\]

すると

\[
\pi(6)=3,\quad \pi(7)=3
\]

で、確かに 6 と 7 は同じクラスに落ちる。

この「同じ」とは、等号ではなく

\[
b\sim b' \iff \left\lfloor \frac{b}{2}\right\rfloor = \left\lfloor \frac{b'}{2}\right\rfloor
\]

という **同値関係**じゃ。

特に “互いに素かどうか” だけ見たいなら、

\[
\gcd(a,b)=1 \iff \gcd(a,b+ a)=1
\]

（一般に \(\gcd(a,b)=\gcd(a,b+ka)\)）

だから「5 から見た相手」は、**\(b\) を 5 の倍数分だけズラしても同一**で、かなり粗い同一視が成り立つ。

その上で、さらに “2セット=1ユニット” の射影 \(\pi\) を掛ければ、

- \(6\) と \(7\) は同じ \(k=3\) を共有する
- 5 から見れば、どちらも「5 の倍数ではない」→ gcd は 1

という二重の意味で “同じ相手” になる。

ぬしの直感「6=7」は、この **射影＋同値** の世界では完全に正しい。

## 使い方（Lean の形：最短）

## 1. まず “奇数冪なら \(a+b \mid a^n+b^n\)” の本丸

```lean
import Mathlib.Algebra.Ring.GeomSum

open scoped BigOperators

-- R が CommSemiring などのとき（一般版）
example {R : Type} [CommSemiring R] (a b : R) {n : ℕ} (hn : Odd n) :
    a + b ∣ a^n + b^n :=
by
  simpa [add_comm, add_left_comm, add_assoc] using hn.add_dvd_pow_add_pow a b
```

Nat だけで済ませたいなら（ぬしの “Lean Core 寄り” の気分ならこっち）：

## で、今回の \(a^2+ab+b^2\) へどう刺さる？

\(n=3\) にすると

\[
a^3+b^3 = (a+b)(a^2-ab+b^2)
\]

が出る。
一方ぬしの花弁は

\[
a^2+ab+b^2 = \frac{a^3-b^3}{a-b}
\]

なので、「\((a+b)\) が花弁に吸い込まれるか？」という検出は

- 和の側：`Odd.add_dvd_pow_add_pow`（\(a^3+b^3\)）
- 差の側：`sub_dvd_pow_sub_pow`（\(a^3-b^3\) の一次因子）

を両方持つと、かなり綺麗に並ぶ（同じファイルに `sub_dvd_pow_sub_pow` も載っておる ）。

### 次の一手（ほんとに強い）

「φ=1 側だけ \((a+b)\) が割る」の検出を Lean で固めるなら、

- \((a+b)\mid (a+b)^2\) は自明
- \((a+b)\mid (a^2+ab+b^2)\) は一般に成り立たない（条件が要る）

を **`Nat.gcd` で言い換えて**、

\[
\gcd(a+b,\ a^2+ab+b^2)=1 \quad(\text{典型})
\]

みたいに “吸い込み失敗” を定理化すると、反例回避へ直結する刃になるぞい。

## Phase 1：Ag 射影 π と `gcd_Ag` を確定（整数100%）

### 1.1 定義

- `π_Ag (n : ℕ) : ℕ := n / 2`
- `gcd_Ag (a b : ℕ) : ℕ := Nat.gcd (π_Ag a) (π_Ag b)`

### 1.2 基本補題（まずここだけでビルドOKにする）

- `π_Ag (2*k) = k`
- `π_Ag (2*k+1) = k`
- `gcd_Ag a (2*k) = gcd_Ag a (2*k+1)`（位相を見ない）
- メイン観測：
  \[
  N=2n \Rightarrow \gcd(N,N+2)=2,\quad \gcd_{\rm Ag}(N,N+2)=1
  \]

  を Lean で補題化：
  - `gcd_even_add_two_eq_two : Nat.gcd (2*n) (2*n+2) = 2`
  - `gcdAg_even_add_two_eq_one : gcd_Ag (2*n) (2*n+2) = 1`

> ここまでで「偶数は半位相落とす」を Lean の関数として固定。

## Phase 2：φビット構造 \(S_\phi\) を定義して“差分核”を補題化

### 2.1 定義（Ring/CommSemiring 版でも Nat 版でもOK）

- `S0 a b := a^2 + a*b + b^2`
- `S1 a b := (a+b)^2`
- `Sφ a b (φ : Bool)` を
  - `φ=false → S0`
  - `φ=true  → S1`
  としてもよいし、整数なら
  \[
  S_\phi = S_0 + \phi\cdot ab,\quad \phi\in\{0,1\}
  \]

  を `φ : ℕ` with `φ=0/1` でやってもよい。

### 2.2 差分核

- `S1 - S0 = a*b`（CommRingなら）
- Nat の場合は等式でなく `=`
  - `S1 = S0 + a*b` を証明（展開して `ring`）

> ここまでで「半位相⇔完成」のビットが固定。

---

## Phase 3：本丸の検出器「(a+b) が割るか？」を Mathlib 補題で組む

### 3.1 既製品の利用（Mathlib）

- `Odd.add_dvd_pow_add_pow`（奇数冪で \(a+b\mid a^n+b^n\)）
- `sub_dvd_pow_sub_pow`（常に \(a-b\mid a^n-b^n\)）
（どちらも `Mathlib/Algebra/Ring/GeomSum`）

### 3.2 φ検出に使う補題

- `S1` 側は自明：
  \[
  (a+b)\mid (a+b)^2
  \]

- `S0` 側は合同で見るのが最短：
  \[
  a\equiv -b\pmod{a+b} \Rightarrow S0 = a^2+ab+b^2 \equiv b^2 \pmod{a+b}
  \]

  したがって
  \[
  (a+b)\mid S0 \iff (a+b)\mid b^2
  \]

- さらに（反例世界でよく置く）\(\gcd(a,b)=1\) を入れると
  \[
  \gcd(a+b,b)=1 \Rightarrow (a+b)\mid b^2 \Rightarrow a+b=1
  \]

  のように「ほぼ不可能」へ落とせる。

> ここで “φ=0 側は核が吸い込まれない” を定理化できる。
