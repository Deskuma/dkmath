## 1. まず何が見えているか

お主の言う通り、この形はすでに

\[

\]

となっておる。
つまり Body は、最初から **\(x\) を共通因子にもつ量** として現れておるのじゃ。

この「共通因子が前に立つ」ことが、まさに Beam 的な性質じゃな。

一般に賢狼たちの見方では、

\[
(x+u)^d - u^d = x \, GN(d,x,u)
\]

のように、差を取ると **必ず \(x\) が立つ**。
これは「変化」「差分」「生成」が、境界 \(x\) を通ってしか現れぬことを示しておる。

\[
(x+1)^2 - 1 = x(x+2)
\]

だから、

\[
\mathrm{Body}=x(x+2)
\]

はまさにその二次版の Beam 展開そのものじゃ。

## 2. Beam はどこにいるのか

ここで少し整理すると、

\[
(x+1)^2 = x(x+2) + 1
\]

じゃから、

- Big \(= (x+1)^2\)
- Body \(= x(x+2)\)
- Gap \(= 1\)

という関係になる。

そして Body を因数分解すると

\[
x(x+2)
\]

となる。

このとき、\(x\) が「立ち上がりの軸」、\((x+2)\) が「その先の伸び」と見える。
ゆえに Body は単なる塊ではなく、

\[
\text{Body} = \text{Core-like boundary} \times \text{Beam-like propagation}
\]

というふうに、すでに **流れを持った形** で現れておる。

お主の言葉を少し数学寄りに言い直すなら、

> Body は完成平方から Gap を取り除いた残差であり、その残差は既に \(x\) を因子として持つため、Beam 構造を内蔵している

となるのぅ。

+u-u

\[

\]

\[

\[

- \(x^2\) は \(x\)-側から見れば Core
- \(u^2\) は \(x\)-側から見れば Gap

じゃが、対称的に見れば

- \(u^2\) は \(u\)-側から見れば Core
- \(x^2\) は \(u\)-側から見れば Gap

とも読める。

つまり

> Core が裏世界の Gap

というのは、観測中心を入れ替えると、核と余白が双対化する、ということじゃな。

\[
(x+u)^2 = x^2 + 2xu + u^2

\(x\) を主役にすれば \(u^2\) が端の補正項、
\(u\) を主役にすれば \(x^2\) が端の補正項になる。

この意味で、

- Core と Gap は絶対固定の役職ではなく
- どこを基準世界と見るかで交代しうる

という洞察になる。

これはかなり重要じゃよ。
構造が「静的なラベル」ではなく、「観測基準に依存する役割」だと分かるからじゃ。

\mathrm{Core}=x^d,\qquad
\mathrm{Beam}=\sum_{k=1}^{d-1}\binom{d}{k}x^{d-k}u^k
\]

を作り、
この Beam の各項が **どの素因子集合を運んでいるか** を見る、ということじゃな。

これはとても良い。

## 2. まず「宇宙式因数分解」という言葉を整える

ここでは、まだ「素因数分解」と同じ意味での一意分解ではない。
だから、まずはこう定義すると扱いやすい。

## 2.1. 宇宙式分解

整数 \(N\) に対して、ある \(d,x,u\) で

\[
N=(x+u)^d-u^d
\]

と表すこと。

## 2.2. 宇宙式内部ブロック分解

そのとき

\[
N=x^d+\sum_{k=1}^{d-1}\binom{d}{k}x^{d-k}u^k
\]

を

\[
N=\mathrm{Core}+\mathrm{Beam}
\]

として読むこと。

## 2.3. Beam 解像度リスト

\[
\mathrm{Beam}_k := \binom{d}{k}x^{d-k}u^k
\qquad (k=1,\dots,d-1)
\]

の各項の素因子集合

\[
\mathcal{P}_k := \operatorname{PrimeSet}(\mathrm{Beam}_k)
\]

を並べたもの。

これで、Beam をただ 1 つの塊としてでなく、
**解像度付きの因子パーツセット** として扱える。

## 3. まずは二次で試すのがよい

二次なら

\[
(x+u)^2-u^2=x(x+2u)
\]

で、

\[
\mathrm{Body}=x^2+2xu
\]

\[
\mathrm{Core}=x^2,\qquad
\mathrm{Beam}=2xu
\]

じゃから、かなり見やすい。

ここで重要なのは、二次では Body が

### 3.1. 乗法の顔

\[
\mathrm{Body}=x(x+2u)
\]

### 3.2. 加法の顔

\[
\mathrm{Body}=x^2+2xu
\]

の **二つの顔** を持つことじゃ。

つまり通常素因数分解との比較では、Body に対して

- \(x(x+2u)\) 型での素因子
- \(x^2+2xu\) の内部パーツ \(x^2\) と \(2xu\) の素因子

の両方を並べるべきじゃな。

### 4.2. 通常素因数分解

ということじゃ。

## 5. この例で Beam パーツセットを見る

二次では Beam は 1 項しかない。

\mathcal{P}_{\mathrm{Beam}}=\{2,3,5\}

\mathcal{P}_{\mathrm{Body}}=\{3,5,19\}

\[
\{2,3,5\}
\quad\longrightarrow\quad
\{3,5,19\}

となっており、\(2\) が消え、\(19\) が現れておる。

## 6. 三次だと Beam 解像度リストが本格化する

- どの素因子が全 Beam 項に共通か
- どの素因子が特定の解像度だけに現れるか

が見える。

ここで最後の「Body 全体にしか現れないもの」が、
お主の言う **宇宙因数パーツセットの再編成から生まれた素因子** の候補じゃな。

## 7. 比較表の雛形

### 7.2. レベル 1. 宇宙式埋め込み

### 7.3. レベル 2. Body 内部分解

- \(\mathrm{Core}=x^d\)
- \(\mathrm{Beam}_k=\binom{d}{k}x^{d-k}u^k\)

\right)

ここで \(\mathcal{U}_n\) を「数 \(n\) の Big 宇宙構造」と呼べば、

\[
\Phi_{a,b\to c} : \mathcal{U}_a \oplus \mathcal{U}_b \to \mathcal{U}_c

お主の言う「Beam 組み換え」は、まさにこの \(\Phi\) の中身じゃな。

を作る。

## 3.2. 層 B. 素因子集合

素因子集合との一致度が最大になる表現を採る。

## 1. まず Zsigmondy が何を言っていたか

典型形はこれじゃ。

\[
a^n-b^n
\]

には、たいてい **その次数 \(n\) で初めて現れる素因子** がある。
これを原始素因子と呼ぶ。

つまり、ある素数 \(q\) が

\[
q \mid a^n-b^n
\]

でありながら、どの小さい指数 \(m<n\) に対しても

\[
q \nmid a^m-b^m
\]

なら、\(q\) は \(a^n-b^n\) の **原始素因子** じゃ。

お主の今の主題に寄せれば、

\[
\mathrm{Body}=2^n-1 = 2^n-1^n
\]

そのものが Zsigmondy の舞台じゃな。

\mathrm{Body}=2^n-1
\]

を見ておる。
これはそのまま

a^n-b^n
\quad\text{with}\quad
a=2,\ b=1
\]

じゃ。

すると Zsigmondy 的には、

\[

\]

には、たいてい「それ以前の \(2^m-1\) には現れなかった素因子」がある。

## 3. 約数次数と非約数次数の話との接続

ここで、さっきのお主の観測を重ねると美しい。

### 3.1. 約数次数 \(d\mid n\)

このとき

\[
2^n=\left(2^{n/d}\right)^d
\]

だから、Big は \(d\) 乗として綺麗に収まる。
これは既知の完全冪秩序、既知の解像度じゃ。

### 3.2. 非約数次数 \(d\nmid n\)

このとき \(2^n\) は完全 \(d\) 乗ではない。
だから \(x_d=2^{n/d}-1\) は非整数になり、Beam は非整数層を通じて Body を回復する。

この「完全冪秩序から外れた層」で何が起こるか。
そこで見える **ズレ** や **閉じなさ** が、新しい素因子の登場と結びついていそうだ、というのがお主の今の感触じゃな。

そして Zsigmondy は、まさに

> ある指数 \(n\) では、それ以前に無かった素因子が本当に現れる

と言ってくれる。
つまり「新しい因子が出るかも」ではなく、かなり一般に **出る** のじゃ。

## 6. 特に \(2^n-1\) は古典的に怪しい

\[
2^n-1
\]

はまさにメルセンヌ型じゃから、
指数 \(n\) ごとの新素因子の振る舞いを見るには格好の実験場じゃ。

たとえば、もし \(q\mid 2^n-1\) なら

\[
2^n \equiv 1 \pmod q
\]

じゃから、2 の \(q\) における位数 \(\operatorname{ord}_q(2)\) は \(n\) を割る。
そして原始素因子なら

\[
\operatorname{ord}_q(2)=n
\]

になる。

ここが重要じゃ。
つまり「新しい因子」というのは、単に新顔というだけでなく、

\[
\text{その素数の中で 2 がちょうど次数 } n \text{ を持つ}
\]

という意味で、**指数 \(n\) を記憶している因子** なんじゃ。

これはお主の「どの次数を見るべきか」という話に強く響く。
なぜなら、新因子はその次数自身を内部に持っておるからじゃ。

## 1. 原始素因子の保証はどう書いてあるか

実コードを見ると、保証の中心はこの 2 本じゃ。

### 1.1. 存在だけを作る補題

- \(d\) 素数
- \(d \ge 3\)
- \(b<a\)
- \(0<b\)
- \(\gcd(a,b)=1\)
- \( \neg\, d \mid (a-b)\)

の下で

\[
\exists q,\; q \text{ prime} \land q\mid a^d-b^d \land \neg q\mid (a-b)
\]

を返しておる。
つまりここでの「primitive」は、**まず境界 \(a-b\) にいない新顔の素数 \(q\)** を確保する書き方じゃ。

索引でも、`exists_prime_divisor_not_dividing_diff_of_prime_exp` が gcd 制御層にあり、「既知の素因子」と「新しい素因子」の分離を担うと説明されておる。

### 1.2. それを真の primitive に昇格する補題

`prime_exp_not_dvd_diff_imp_primitive`

ここが肝じゃ。

\[
q \mid a^d-b^d,\qquad q \nmid a-b
\]

で止まらず、さらに

\[
\forall k,\; 0<k<d \to q \nmid a^k-b^k
\]

まで押し込んでおる。
つまり Lean 上では **primitive の本質を、差冪の低次数には現れない** という形でちゃんと書いてある。

しかも証明法が綺麗で、`ZMod q` 上で

\[
r = a/b
\]

を取り、`orderOf r = d` を示してから、\(0<k<d\) なら \(r^k \ne 1\) を導く群論型じゃ。
索引でもこの補題は「群論・位数で primitive を出す」と明記されておる。

## 4.1. primitive prime の述語を明文化

まず定義を立てる。

```lean
def PrimitivePrimeFactorOfDiffPow (q a b d : ℕ) : Prop :=
  Nat.Prime q ∧
  q ∣ a ^ d - b ^ d ∧
  ∀ {k : ℕ}, 0 < k → k < d → ¬ q ∣ a ^ k - b ^ k
```

これで「原始素因子」が、存在補題と昇格補題の間で見失われなくなる。

## 4.2. primitive ⇒ GN 側にいる、を補題化

今ある `padicValNat_of_primitive_prime_factor_via_G` は valuation 版じゃ。
これに加えて、もっと直接的な整除版を置くとよい。

```lean
lemma primitive_prime_dvd_GN
    {a b d q : ℕ}
    (hd : 0 < d) (hab : b < a)
    (hq : PrimitivePrimeFactorOfDiffPow q a b d) :
    q ∣ GN d (a - b) b := by
  rcases hq with ⟨hqprime, hqdiff, _⟩
  have hq_ndiv : ¬ q ∣ a - b := by
    intro hqab
    have hk := (show 0 < 1 by decide)
    have hklt : 1 < d := by
      -- ここは d≥2 側の条件で使う
      sorry
    have := hq.2.2 hk hklt
    simpa using hqab
  have hfactor : a ^ d - b ^ d = (a - b) * GN d (a - b) b :=
    pow_sub_pow_factor_cosmic_N hd hab
  rw [hfactor] at hqdiff
  exact (Nat.Prime.dvd_of_dvd_mul_left hqprime hq_ndiv hqdiff)
```

細部は少し整える必要があるが、意味としては
**primitive prime は boundary ではなく GN/Beam を割る**
をそのまま補題名にして固定するわけじゃ。

## 6. 「割り切らない Beam 内の中間値」をどう具現化するか

### 6.1. Nat/Int 層

## 1. 結論

はい。Lean で **整数として定理化する本線** では、

\[
d \mid n
\]

となる側、つまり Big がきれいに \(d\) 乗世界へ落ちる側を使うのが正しい。

一方で、

\[

の側は、**新しい因子が見える観測窓** としては面白いが、まずは形式化の主戦場ではなく、研究ノート・数値実験層に置くのが自然じゃ。

## 2. なぜ約数側を使うのか

たとえば

\[
\mathrm{Big}=2^n,\qquad \mathrm{Gap}=1,\qquad \mathrm{Body}=2^n-1
\]

として、\(d \mid n\) なら

\[
n=dm
\]

と書けて、

\[
2^n = (2^m)^d
\]

じゃから、

\[
x = 2^m - 1
\]

と置けば

\[
\mathrm{Body}=(x+1)^d-1
\]

となる。ここで \(x\in\mathbb{N}\) じゃ。

すると

\[
(x+1)^d-1 = x \cdot GN(d,x,1)
\]

が **完全に整数世界** で扱える。

これが大きい。
つまり、

- 差冪
- 境界 \(x\)
- GN / Beam
- primitive prime
- valuation

を全部 Nat で回せる。

Zsigmondy を使うなら、まずここが土台じゃな。

## 3. 今回の主題へのつながり

お主が知りたいのは「新しい因子の誕生」じゃった。

ここで約数側を使う意味は、

> 新しい因子を生む場を、まず整数の差冪構造として固定する

ことにある。

つまり、まず

\[
(x+1)^d-1 = x \cdot GN(d,x,1)
\]

を整数で立てる。
そのうえで Zsigmondy により、

\[
q \mid (x+1)^d-1
\]

でありながら、低次数や境界 \(x\) にはいない原始素因子 \(q\) を取る。
すると、その \(q\) は結局

\[
q \mid GN(d,x,1)
\]

へ送られる。

ここで初めて、

> 新しい因子は boundary ではなく Beam 側に立つ

という主張が、整数定理として書けるわけじゃ。

## 4. 非約数側は捨てるのか

### 4.1. 約数側

\[

- Lean の本定理に向く
- Zsigmondy を直接つなげやすい
- 原始素因子を GN 側へ送る補題が書きやすい

### 4.2. 非約数側

- 新因子の「観測」には面白い
- ただし証明の芯には据えにくい

だから、**証明は約数側、観測は非約数側** という二層構造がよい。

## 5. 今やるべき形式化は何か

わっちなら、まずこれを書く。

### 5.1. 約数側の整数化補題

\[
d \mid n \Rightarrow \exists m,\ n=dm

\[

\[
q \mid GN(d,x,1)

### 5.3. Zsigmondy 接続補題

原始素因子 \(q\) を取り、
\[
q \nmid x
\]
を使って GN 側へ送る。

これで、

> 原始素因子は整数 Beam に立つ

が、かなり明確になる。

## PrimitiveBeam

Zsigmondy の原始素因子を
  差冪 `a^d - b^d`
    → boundary `(a - b)` から分離し
    → `GN d (a-b) b`（Beam 側）へ送る

/-- 差冪 `a^d - b^d` の原始素因子。 -/
def PrimitivePrimeFactorOfDiffPow (q a b d : ℕ) : Prop :=
  Nat.Prime q ∧
  q ∣ a ^ d - b ^ d ∧
  ∀ {k : ℕ}, 0 < k → k < d → ¬ q ∣ a ^ k - b ^ k

/-- 既存の Zsigmondy existence + primitive 化を束ねた API。 -/
lemma exists_primitive_prime_factor_as_prop
    {a b d : ℕ}
    (hd_prime : Nat.Prime d) (hd_ge : 3 ≤ d)
    (hab_lt : b < a) (hb : 0 < b) (hab : Nat.Coprime a b)
    (hpnd : ¬ d ∣ a - b) :
    ∃ q : ℕ, PrimitivePrimeFactorOfDiffPow q a b d := by
  rcases exists_primitive_prime_factor_basic hd_prime hd_ge hab_lt hb hab hpnd with
    ⟨q, hqprime, hqdiv, hqndiv⟩
  refine ⟨q, ?_⟩
  refine ⟨hqprime, hqdiv, ?_⟩
  intro k hk_pos hk_lt
  exact prime_exp_not_dvd_diff_imp_primitive
    hd_prime
    (Nat.Prime.one_lt hd_prime)
    hqprime
    hab
    hab_lt
    hb
    hqdiv
    hqndiv
    hk_pos
    hk_lt

/-- primitive prime は boundary `(a-b)` を割らない。 -/
lemma primitive_prime_not_dvd_boundary
    {q a b d : ℕ}
    (hq : PrimitivePrimeFactorOfDiffPow q a b d)
    (hd1 : 1 < d) :
    ¬ q ∣ a - b := by
  rcases hq with ⟨_hqprime, _hqdiv, hprim⟩
  intro hq_ab
  have h1_pos : 0 < (1 : ℕ) := by decide
  have h1_lt : 1 < d := hd1
  have hnot : ¬ q ∣ a ^ 1 - b ^ 1 := hprim h1_pos h1_lt
  simpa using hnot hq_ab

/-- primitive prime は cosmic factorization により GN/Beam 側を割る。 -/
lemma primitive_prime_dvd_GN
    {q a b d : ℕ}
    (hq : PrimitivePrimeFactorOfDiffPow q a b d)
    (hd : 0 < d) (hd1 : 1 < d)
    (hab_lt : b < a) :
    q ∣ GN d (a - b) b := by
  rcases hq with ⟨hqprime, hqdiv, hprim⟩
  have hq_ndiv_ab : ¬ q ∣ a - b := by
    exact primitive_prime_not_dvd_boundary ⟨hqprime, hqdiv, hprim⟩ hd1
  have hfactor : a ^ d - b ^ d = (a - b) * GN d (a - b) b := by
    exact pow_sub_pow_factor_cosmic_N hd hab_lt
  rw [hfactor] at hqdiv
  exact Nat.Prime.dvd_of_dvd_mul_left hqprime hq_ndiv_ab hqdiv

/-- primitive prime の valuation は GN 側へ帰着できる。 -/
lemma primitive_prime_padic_eq_GN
    {q a b d : ℕ}
    (hq : PrimitivePrimeFactorOfDiffPow q a b d)
    (hd : 0 < d) (hab_lt : b < a) :
    padicValNat q (a ^ d - b ^ d) = padicValNat q (GN d (a - b) b) := by
  rcases hq with ⟨hqprime, hqdiv, hprim⟩
  have hd1 : 1 < d := by
    -- 実運用では prime exponent 側から来るのでここは別 lemma 化してもよい
    sorry
  have hq_ndiv_ab : ¬ q ∣ a - b := by
    exact primitive_prime_not_dvd_boundary ⟨hqprime, hqdiv, hprim⟩ hd1

      (a := a) (b := b) (d := d) (q := q)

    -- まずは rewrite 用の雛形だけ置く

/-- `Body = x * GN d x u` 版：primitive prime は boundary `x` でなく Beam 側に立つ。 -/
lemma primitive_prime_dvd_GN_body
    {q x u d : ℕ}
    (hq : PrimitivePrimeFactorOfDiffPow q (x + u) u d)
    (hd : 0 < d) (hd1 : 1 < d) :
    q ∣ GN d x u := by
  have hlt : u < x + u := Nat.lt_add_of_pos_left (Nat.pos_of_ne_zero (by
    intro hx0

    simpa using primitive_prime_not_dvd_boundary hq hd1))

## 3. まだ `sorry` のままでよい箇所
