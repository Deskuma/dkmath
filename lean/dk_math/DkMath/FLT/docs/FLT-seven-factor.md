# 研究メモ：`a^2+ab+b^2` は「平方に足りない `ab`」だが「三角格子ノルムとして閉じる」

see: [議論#260223-2027](/lean/dk_math/notes/Agent-note-260223-2027.md)

## 0. 目的（この文書の使い方）

本メモは、FLT3 で反例側に現れる `S(a,b) := a^2+ab+b^2` を、

- (A) 「`(a+b)^2`（直交格子の平方）に届く直前で `ab` が不足している」構造
- (B) 「アイゼンシュタイン（正確には 60°格子）ノルムとして閉じている」構造

として整理し、Lean 形式化で必要になる補題候補を列挙する。

最終ゴールは：

- 既存ソース（`PetalDetect`, `GcdNext`, `GEisensteinBridge`, `DiffPow`, `GcdDiffPow` 等）と突き合わせ、
- “不足補題” をチェックリスト化し、追加実装に繋げること。

---

## 1. 基本定義と恒等式（確定事項）

### 1.1 反例側に現れる中間式

`S(a,b)` を

- `S(a,b) := a^2 + a*b + b^2`

と定義する。

これは FLT3 の因数分解

- `a^3 - b^3 = (a-b) * (a^2 + a*b + b^2)`  （整数環での恒等式）

の “第2因子” として自然に現れる。

> Lean では `ℤ` 上で `ring_nf` / `ring` により即証明でき、`ℕ` へは `zify` 等を通すか `Nat` の差分条件 `b ≤ a` などで整える。

---

### 1.2 「平方に足りない `ab`」恒等式

以下が核：

- `(a+b)^2 = (a^2 + a*b + b^2) + a*b`

従って

- `S(a,b) = (a+b)^2 - a*b`

差分として

- `(a+b)^2 - S(a,b) = a*b`

が成り立つ。

これは「`S(a,b)` は平方に届く直前。足りない量は `ab`」という意味。

> 後段の p-adic / gcd 議論で、`ab` が“吸収できない補正”として効く。

---

## 2. 「三角で閉じる」＝ 60°格子（Eisenstein/hexagonal）ノルム

### 2.1 ノルム表現（基底の取り方に注意）

`S(a,b)=a^2+ab+b^2` は、60°格子の代数整数 `ζ6 = exp(iπ/3)` を用いると

- `N(a + b*ζ6) = a^2 + a*b + b^2`

となるタイプのノルム（実数値の二次形式）として現れる。

※一般に “アイゼンシュタイン整数” と言うと `ω=exp(2πi/3)` を使い `a^2-ab+b^2` がノルムに出ることが多い。
本研究の `+ab` 版は「同じ体（Q(√-3)）だが基底の取り方が異なる」もの、と整理しておくと混線が減る。

> Lean では複素数体まで持ち込まずとも、二次形式 `a^2 + ab + b^2` の代数的性質（合同条件、素因子分類、gcd）だけ抽出して進めるのが現実的。

---

## 3. FLT3 観点：`S(a,b)` が「きれいな平方」にならない方向づけ

### 3.1 自明だが重要な観察（“補正で平方化できる”）

`S(a,b)+ab` は常に平方：

- `S(a,b) + a*b = (a+b)^2`

従って「`q=(a+b)` で `q^2` が得られる」のは tautology（恒等式）。

重要なのは逆：

- `S(a,b)` 単体では一般に `q^2`（特に `(a+b)^2`）で割れない  
  （`S = q^2 - ab` のため）

ここから “`ab` が吸収できない” 条件を gcd で与えると、`S` が n 乗（特に平方）になる道が塞がれる。

---

## 4. gcd / 位相（mod 3）に関する典型事実（Lean 補題候補）

ここが形式化の主戦場になる。以下、確度順に。

### 4.1 まず確実に欲しい：mod 3 の基本合同

#### (L1) `a^2+ab+b^2` の mod 3 判定（平方剰余の分類）

`a ≡ b (mod 3)` なら `S(a,b) ≡ 0 (mod 3)` が起きやすい。

実際

- `S(a,b) = (a-b)^2 + 3ab` という恒等式がある：

`(a-b)^2 = a^2 - 2ab + b^2` なので
`(a-b)^2 + 3ab = a^2 + ab + b^2 = S(a,b)`.

従って

- `S(a,b) ≡ (a-b)^2 (mod 3)`。

よって `3 ∣ S(a,b)` ⇔ `3 ∣ (a-b)`（少なくとも整数環の合同としてはこの方向が見える）。
ただし「⇔」は `mod 3` での平方の性質を踏む必要があるので、Lean では

- `S ≡ (a-b)^2 [ZMOD 3]` の補題
- `x^2 ≡ 0 [ZMOD 3] ↔ 3 ∣ x`（Nat/Int どちらでやるか注意）
が要る。

#### (L2) `S(a,b) = (a+b)^2 - ab` の mod 3 判定

`ab` の 3 進位相が、平方化を妨げる「足りない補正」として働く。
このとき

- `3 ∣ ab` か否かで場合分け（Phase 2/3 的）を作れる。

---

### 4.2 gcd 系：`gcd(a,b)=1` なら `gcd(a, S)=1` など

典型的に欲しい補題群（※正確な形は既存実装に合わせて調整）

#### (G1) `Nat.Coprime a b → Nat.Coprime a (a^2+ab+b^2)`

理由：`S ≡ b^2 (mod a)` なので `gcd(a,S)=gcd(a,b^2)=1` が期待できる。

同様に

#### (G2) `Nat.Coprime a b → Nat.Coprime b (a^2+ab+b^2)`

> これらは “`S` に a や b の素因子が混ざりにくい” を保証し、Zsigmondy/p-adic の分岐条件作りに効く。

#### (G3) `Nat.Coprime a b → gcd (a+b) b = 1` のような補助（Phase 3）

既に `PetalDetect` に近い形があるなら、それを使って `S` に対して

- `(a+b) ∣ S ↔ (a+b) ∣ b^2`  
の類を引き回す（ぬしの既存補題に合流）。

---

### 4.3 “三角ノルム素因子分類”への入り口（必要なら）

解析を深めるなら、以下が欲しくなる可能性が高い：

#### (E1) `p ≡ 2 (mod 3)` の素数は `a^2+ab+b^2` を割るなら強い制約を与える

古典的には、`Z[ω]` の分解法則（`p ≡ 1 mod 3` は分解、`p ≡ 2 mod 3` は惰性、`p=3` は分岐）に対応し、
`a^2+ab+b^2` の表現可能性・割り切り可能性が mod 3 クラスで分かれる。

Lean で全部やるのは重いので、最初は

- **必要最小限の合同補題**（(L1)〜）
- **gcd 制御**（(G1)(G2)）
- **p-adic の平方性否定**（既存の `hS0_not_sq` 型）

で押し切るのが現実的。

---

## 5. Lean 実装に落とすときの “型” と “戦術” の指針

### 5.1 `ℤ` に上げて `ring_nf` → `ℕ` へ戻す

- 因数分解・恒等式：`ℤ` 上で `ring_nf` が安定。
- `ℕ` 上の引き算や差：`zify` と `b ≤ a` 等が要る（既に会話で確立した手）。

**候補テンプレ**

- `have h : (a:ℤ)^3 - (b:ℤ)^3 = ... := by ring_nf`
- `zify [..]` で Nat の式へ戻す

### 5.2 `S(a,b)` を専用定義にして “引き回し” を簡単にする

`def S0 (a b : ℕ) := a^2 + a*b + b^2` のような定義を一箇所に置き、

- `simp [S0]` で展開
- `have : S0 a b = (a+b)^2 - a*b := by ring`（ℤで）
などを “標準ライブラリ” にする。

## 6. 不足補題チェックリスト（このまま TODO 化）

### 6.1 恒等式（必須）

- [ ] `S = (a+b)^2 - ab`（Nat/Int どちらで保持するか）
- [ ] `S = (a-b)^2 + 3ab`（mod 3 用の形）

### 6.2 合同（mod 3）

- [ ] `S ≡ (a-b)^2 [ZMOD 3]`
- [ ] `x^2 ≡ 0 [ZMOD 3] ↔ 3 ∣ x`（Nat/Int）
- [ ] `3 ∣ S ↔ 3 ∣ (a-b)`（導出系）

### 6.3 gcd 制御

- [ ] `Coprime a b → Coprime a S`
- [ ] `Coprime a b → Coprime b S`
- [ ] `Coprime a b → gcd (a+b) a = 1` / `gcd (a+b) b = 1`（Phase 条件用）
- [ ] 既存補題 `(a+b) ∣ S ↔ (a+b) ∣ b^2` 系との接続（PetalDetect）

### 6.4 p-adic の平方否定（既存仮定と繋ぐ）

- [ ] `q ∣ (c^3-b^3) ∧ ¬ q ∣ (c-b) → ¬ q^2 ∣ S0 c b` の形の “構築側” を、上の gcd/mod3 補題から生むルートを決める

---

## 7. まとめ（本メモの要点）

- `S(a,b)=a^2+ab+b^2` は `ab` を足せば平方になる：`S+ab=(a+b)^2`。
- しかし `S` 単体は “平方直前の中間層” として残り、ここが FLT3 の反例側に現れる核心。
- `S` は 60°格子（Eisenstein/hexagonal）的にはノルムとして閉じるため、「直交平方に閉じないのに三角で閉じる」という幾何直感は正しい。
- Lean での勝ち筋は、複素数体を持ち込むより
  - 恒等式（2形）
  - mod 3 合同
  - gcd 制御
  を固め、既存の Phase / padicValNat / Zsigmondy の仮定・補題と連結すること。

以上。
