# GKUS 設計総括

## 1. 概要

本稿は、`KUS` 系の係数一般化として導入された `GKUS` の設計を、形式化実装（Lean 4）と数学的意図の両面から総括する。

`GKUS` の核心は次の 1 点に尽きる。

- 係数計算（`Nat`, `Int`, `Rat`, …）と、構造保持（`US`）を分離し、演算後も support を必ず追跡可能にする。

この分離により、通常の数体系で結果が `0` に潰れて失われる情報を、`extract_g` によって回収できる。

## 2. 問題設定

従来の `KUS` は係数型が `Nat` 固定であり、加法・乗法の枠組みは十分であったが、次の制約があった。

1. 減法を自然に扱えない。
2. 分数係数（有理数）を直接表せない。
3. 除算とその代数関係を統一的に扱いにくい。

この制約を解消するため、係数型を任意の `C` に一般化した `GKUS C U Blueprint` を導入した。

## 3. 形式モデル

### 3.1 基本構造

`GKUS` は次の 3 成分を持つ。

- `coeff : C`
- `unit : U`
- `blueprint : Blueprint unit`

support は

- `extract_g : GKUS C U Blueprint → US U Blueprint`

で抽出され、係数と独立に保持される。

### 3.2 同一 support 述語

- `GSameSupport x y : Prop := extract_g x = extract_g y`

を導入し、すべての二項演算はこの前提の下で定義する。

### 3.3 演算核

演算は基底演算

- `gOp (op : C → C → C)`

として抽象化し、

- `gAdd`（加算）
- `gMul`（乗算）
- `gSub`（減算）
- `gDiv`（除算）

を `gOp` の特殊化として与える。

この設計により、証明は「係数側の演算法則」と「support 側の保持法則」に分離できる。

## 4. 代数階層と法則

係数型 `C` に要求する型クラスは段階的に整理した。

1. 加法交換: `[AddCommMonoid C]`
2. 加法結合: `[AddSemigroup C]`
3. 乗法交換: `[CommMonoid C]`
4. 乗法結合: `[Semigroup C]`
5. 分配法則: `[Distrib C]`
6. 除算法則（本稿追加）: `[DivisionRing C]`

`gDiv` 関連で固定した法則は次の 3 本である。

1. `gDiv_one`: `x / 1 = x`
2. `gDiv_add_distrib`: `(x + y) / z = x/z + y/z`
3. `gMul_gDiv_assoc`: `x * (y / z) = (x * y) / z`

## 5. 証明様式

本実装の安定化に寄与した証明パターンは次の通り。

1. 交換法則: `simp` と `rw` の最短構成。
2. 結合法則・分配法則: `GKUS.ext` で 3 フィールドへ分解。
3. 係数側: `simp [gOp, add_assoc, mul_assoc, add_mul, mul_add, add_div, div_one, ...]`。
4. support 側: `simp [gOp]` のみで閉じる。

特に 4. は、`GKUS` 理論の主張である「演算結果にかかわらず構造保持が失われない」を、証明レベルで直接表している。

## 6. Rat 係数検証

`DkMathTest` では `Rat` 係数値

- `grA = 1/2`
- `grB = 1/3`
- `grC = 1/4`

を使い、加法・乗法・減法・除算とその代数法則を回帰テスト化した。

これにより、`Nat` 固定設計では扱いにくかった分数係数の文脈で、`GKUS` 設計が一貫して動作することを確認した。

## 7. 理論的意義

`GKUS` は次の 2 層を分離する。

1. **観測層（係数）**: 通常の代数計算。
2. **構造層（support）**: 失われるべきでない情報の保持。

この二層分離により、「値が 0 になった」という事実と「何が 0 に潰れたか」を同時に記述できる。

## 8. 制約と今後

現段階の制約は次の通り。

1. 二項演算は `GSameSupport` 前提のもとで定義される。
2. 異なる support 間演算の正準化（transport 規則）は未導入。
3. `DivisionRing` 以上を必要とする除算法則は、係数型の適用範囲を選ぶ。

今後の課題としては、次を想定する。

1. 異 support 演算の整合法則（transport の公理化）。
2. `GKUS` の構造をより高階の代数オブジェクトとして再編。
3. KUS/GKUS を用いた数論的構造追跡問題への具体適用。

## 9. 結語

`GKUS` によって、KUS 系は `Nat` 固定の試作段階から、係数一般化と除算法則までを備えた拡張段階へ進んだ。

本設計の本質は、

- 係数演算の一般性
- support 追跡の不変性

を同時に満たす点にある。これは、零化による情報消失を扱う新しい形式的数論基盤として、今後の拡張に耐える最小核である。
