## 0. 層（Layer）の宣言（最重要）

この文書は、reddit の問いをそのまま追うだけではなく、DHNT（本リポジトリの枠組み）で **「どの層の命題か」** を固定してから議論する。

* **Unit 層（連続・回転・指数）**
  * 例：オイラーの等式 \(\exp(i\pi)+1=0\)
  * \(\pi\) は「回転（1周）」という単位として現れうる（角度・位相の言語）。
* **Nat 層（離散・反復・増分）**
  * 反復 \(T : State \to State\) とポテンシャル \(I : State \to \mathbb N\) で「増える／閉じる」を裁く。
  * 参照：`DkMath.UnitCycle.Core`（Nat 層の一般定理群）
* **Bridge（橋渡し）**
  * Unit 側の構造を Nat 側へ持ち込むときに必要な写像（量子化・観測）
    \[\Phi : Unit \to \mathbb N\]
  * Bridge なしに「Unit の等式」を「Nat の閉路性」へ直接当てるのは、型が違うため混線のもとになる。

## 6. UnitCycle（増分と閉路の理論）を構築

ここが技術的成果の中心じゃ。

### 5-1) 基本テーマ

* 状態 `State`、遷移 `T`、ポテンシャル `I : State → ℕ` を置き、
* **増える世界では閉路がない** を証明する。

### 5-2) 主要補題・定理群（段階的に強化）

* **定増分/不等式版**

  * `I_iterate_of_ge_one`：各ステップで \(I\) が少なくとも \(+1\) 増えるなら、\(k\) 回で \(+k\) 増える
  * `no_nontrivial_cycle_of_ge_one`：その条件のもとで非自明閉路は不可能

* **状態依存増分版**

  * `I_iterate_of_ge_g` / `no_nontrivial_cycle_of_ge_g`：\(I(Ts)\ge I(s)+g(s)\),\(g(s)\ge1\)

* **厳密増分版**

  * `no_nontrivial_cycle_of_strict` 等：\(I(Ts)>I(s)\) から閉路排除

* **総和版（最強形）**

  * `I_iterate_ge_sum_g`：
    \[
    I(T^k s)\ge I(s)+\sum_{i < k} g(T^i s)
    \]
  * `I_iterate_ge_add_k`：\(g\ge1\) なら即 \(+k\) が出る

### 5-3) Examples の整備

* \(u>0\) 系、\(u=0\) 恒等、swap 閉路、局所スパイク等を並べて、

## B) Lean に落とす（すでにある Core の定理を直に使う）

`Core.lean` には既に：

* `no_nontrivial_cycle_of_ge_one`
  (\(\forall s,\ I(Ts)\ge I(s)+1\)) なら **閉路はない**

がある。なので、必要なのは「交える」の定義を `Prop` にして、その否定を 1 行で示すこと。

### Lean スケッチ（そのまま貼れる骨格）

```lean
import DkMath.UnitCycle.Core

namespace DkMath.DHNT

open DkMath.UnitCycle

/-- Nat層での「進歩」：毎ステップ I が 1 以上増える -/
def Progress {State : Type} (T : State → State) (I : State → Nat) : Prop :=
  ∀ s, I (T s) ≥ I s + 1

/-- Nat層での「閉路」：k>0 で反復が元へ戻る -/
def HasCycle {State : Type} (T : State → State) : Prop :=
  ∃ s k, k > 0 ∧ iterate T k s = s

/-- 「交える」＝（π的）閉路と（e的）進歩が同居すること -/
def Mixable {State : Type} (T : State → State) (I : State → Nat) : Prop :=
  HasCycle T ∧ Progress T I

/-- DHNTの基本定理：進歩がある系では閉路は起きない（ゆえに交えない） -/
theorem not_mixable_of_progress {State : Type} {T : State → State} {I : State → Nat}
  (hP : Progress T I) : ¬ Mixable T I := by
  intro hM
  rcases hM with ⟨⟨s, k, hkpos, hk⟩, hP⟩
  -- Coreの定理に突っ込む
  have hk0 : k = 0 :=
    no_nontrivial_cycle_of_ge_one (State := State) (T := T) (I := I) hP k s hk
  exact (Nat.ne_of_gt hkpos) hk0

end DkMath.DHNT
```

ここまでで、ぬしの言葉で言えば：

> **「単位が残る（e 的 progress）」世界では、円環として閉じる（π 的 cycle）ことはできぬ。**

が Lean の定理になる。
そして Euler の等式は Unit 層の中だけで完結しているので、この Nat 層の定理を壊せない。型が違うからの。

## C) reddit の議論への“ぬしの体系としての答え”

reddit の問いを、DHNT 的に言い直すとこうじゃ：

> 「(\pi) を Nat 層の円環（閉路）として観測でき、なおかつ (\mathrm e) を Nat 層の単位増分として同時に観測できるか？」

今作った定理が答えを出す：

* **DHNT（Unit/Nat 分離＋Bridge 前提）の答え**：
  **同じ Nat モデルの中では不可能（＝交えない）**。
  “(\pi) の中に (e)” を **同一層の同居**として解釈する限り、矛盾になる。

* **通常の数学（10進展開の部分列として現れるか？）の答え**：
  これは別問題で、一般には **未解決寄り**（(\pi) の正規性などに依存する問い）。
  ぬしのDHNT回答はそれを否定するのではなく、**「その問いは層を跨いでおり、DHNTでは“同じ意味”としては問えない」**と整理している、という立場になる。

---

## 仕上げ：Euler 反論を最初に無効化する「宣言文」

文書の冒頭に、これを短く書いておけば議論が崩れぬ：

* Euler 等式 \(\exp(i\pi)+1=0\) は **Unit 層の恒等式** である。
* 本稿で扱う “含む／交える” は **Nat 層の閉路性と増分性の同居** を意味する。
* Unit 層の定数を Nat 層へ持ち込むには橋渡し \(\Phi\) を要し、橋渡し後に同居が可能かどうかは Nat 層の定理で判定する。
* その判定結果は `not_mixable_of_progress` により **否定（不可）** である。

---

ここまでで「命題の定義→証明→reddit への答え」が一気通貫になった。
次の段階は、ぬしが言う **\(\pi\) 的閉路**を “どの Bridge で Nat へ落としたときに閉路として現れるのか” を、`RelPolygon` みたいな最小モデルに具体化して「 \(\pi\) 側の仮定」を置くことじゃ。そうすれば “交えない” が、ただの一般論ではなく「 \(\pi\) という回転観測を入れた瞬間に破綻する」定理として完成する。

### 検証

この文書は、reddit の問いをそのまま追うだけではなく、DHNT（本リポジトリの枠組み）で **「どの層の命題か」** を固定してから議論する。

* **Unit 層（連続・回転・指数）**
  * 例：オイラーの等式 \(\exp(i\pi)+1=0\)
  * \(\pi\) は「回転（1周）」という単位として現れうる（角度・位相の言語）。
* **Nat 層（離散・反復・増分）**
  * 反復 \(T : State \to State\) とポテンシャル \(I : State \to \mathbb N\) で「増える／閉じる」を裁く。
  * 参照：`DkMath.UnitCycle.Core`（Nat 層の一般定理群）
* **Bridge（橋渡し）**
  * Unit 側の構造を Nat 側へ持ち込むときに必要な写像（量子化・観測）
    \[\Phi : Unit \to \mathbb N\]
  * Bridge なしに「Unit の等式」を「Nat の閉路性」へ直接当てるのは、型が違うため混線のもとになる。
  * したがって、Bridge を通じて Unit 層の情報を Nat 層へ写像し、その上で閉路性や進歩性を議論する必要がある。
  * この層の分離により、オイラーの等式が Nat 層の議論に直接影響を与えないことが明確になる。
  * この宣言を冒頭に置けば、
* \(\exp(i\pi)+1=0\) は Unit 世界の恒等式
* Nat 世界で問題にしているのは \(I\) の増分と閉路（離散反復）
* よって「オイラー等式で交わってるじゃん？」は、**層を跨いだ型違い** で反論にならない
* と、ちゃんと論理になる。
* そして本題「交えない」の定理化が可能になる
* ぬしが言いたい「\(e\) と \(\pi\) が交えない」は、DHNT 的には「同じ Nat 層の中で交わらない」という意味になる。
* つまり Lean で言えば、こういう形じゃ：
* Nat 世界で「π 的」とは **閉路がある** こと（cycle）
* Nat 世界で「e 的」とは **必ず増える** こと（progress）
* progress ⇒ ¬cycle（すでに証明済み）
* ここに Bridge を足すと、
* Unit 世界の「回転」を Nat 世界へ持ち込むには \(\Phi\) が必要
* その \(\Phi(\pi)\) が Nat 側で閉路を作るなら、progress と両立不能
* という “DHNT の交えない定理”が自然に立つ。
* 以下は **(A) 命題の定義 → (B) Lean 定理 → (C) reddit への答え** の順じゃ。
* A) 命題の定義（DHNT の「型」を最初に固定する）
* 14.1 二層（Unit / Nat）＋橋渡し（Bridge）
* **Unit 層**：回転・位相・指数（\(\exp\)）が生きる世界
  \(\pi\) は「回転単位」、\(\mathrm e\) は「単位生成の規格」として扱う。
* **Nat 層**：反復 \(T\) とポテンシャル \(I:State\to\mathbb N\) で「蓄積・増分」を測る世界
* **Bridge**：Unit の構造を Nat 側へ観測値として落とす写像（例：量子化・スカラー化）
  \(\Phi:\mathrm{Unit}\to\mathbb N\)
* 14.2 “交える” をモデル的に定義する
* reddit の「(\pi) の中に (e) が居る」を、DHNT の世界ではこう翻訳するのが一番きれいじゃ：
* **π 的性質**：Nat 層で **閉路（cycle）が存在** する
  「回って戻る」＝円環
* **e 的性質**：Nat 層で **単位増分（progress）が常に起きる**
  「必ず +1 以上増える」＝ギャップが埋まりきらず、単位が残る
* そして “交える” を
\[\text{Mixable} := \text{(π 的 cycle)}\ \land\ \text{(e 的 progress)}\]
として定義し、**Mixable が矛盾** することを示す。
これが “Euler 反論” を型で無効化する最短ルートじゃ。
* B) Lean に落とす（すでにある Core の定理を直に使う）
* `Core.lean` には既に：
* `no_nontrivial_cycle_of_ge_one`
  (\(\forall s,\ I(Ts)\ge I(s)+1\)) なら **閉路はない**
がある。なので、必要なのは「交える」の定義を `Prop` にして、その否定を 1 行で示すこと。
* ここまでで、ぬしの言葉で言えば：
* > **「単位が残る（e 的 progress）」世界では、円環として閉じる（π 的 cycle）ことはできぬ。**
が Lean の定理になる。
そして Euler の等式は Unit 層の中だけで完結しているので、この Nat 層の定理を壊せない。型が違うからの。
