G_{d-1}(x,u)\equiv d\,u^{d-1}\pmod{x}.

## 2. まず最重要の事実

## A. d が素数なら中間係数は全部 d で割れる

これは超重要じゃ。

もし \(p\) が素数なら

\[
\binom{p}{k}=
\frac{p!}{k!(p-k)!}
\qquad (1\le k\le p-1)
\]

で、分母の \(k!\) と \((p-k)!\) には \(p\) が入らぬ。
一方、分子 \(p!\) には \(p\) が 1 回だけ入る。
したがって

\[
p \mid \binom{p}{k}
\qquad (1\le k\le p-1)
\]

となる。

つまり

\[
(x+u)^p=
x^p+u^p
+
p\sum_{k=1}^{p-1}\frac{1}{p}\binom{p}{k}x^k u^{p-k}
\]

であり、Beam 全体に \(p\) が因子として出る。

これは数論ではとても大きい。
なぜなら

\[
(x+u)^p \equiv x^p+u^p \pmod p
\]

が従うからじゃ。
これは「Freshman's dream」と呼ばれることもある。名前はちょっと間抜けじゃが、中身は強い。

## 3. 逆に d が合成数だと何が起きるか

ここが Beam の素数性観測の核心じゃな。

たとえば \(d=6\) では

\[
\binom{6}{1}=6,\quad
\binom{6}{2}=15,\quad
\binom{6}{3}=20.
\]

このうち

\[
15,\ 20
\]

は \(6\) で割れぬ。
つまり「Beam 全体に 6 が共通因子として出る」ことはない。

ここが **素数指数と合成指数の決定的な差** じゃ。

ぬしの感覚で言えば、

- 素数次数 \(p\) の Beam は、全体が \(p\) という一本の軸で束ねられる

という感じじゃな。
実に建築学的でおもしろい。

## 4. なぜ二項係数に素数が現れるのか

\[

つまり

- 下側の \(k!\) が打ち消す素因数
- 打ち消し切れず残る素因数

## 6. Lucas と Kummer の定理

## B. 二項係数の素数性は p 進表示に支配される

ここから少し深みに降りるぞ。実にうまい話じゃ。

### Kummer の定理

素数 \(p\) に対して、

\[
v_p\!\left(\binom{n}{k}\right)
\]

は、\(k+(n-k)\) を \(p\) 進法で足すときに生じる **キャリーの回数** に等しい。

これはべらぼうに美しい。
ぬしが前から興味を持っておる **キャリー** と二項係数が、ここで直結する。

つまり Beam の係数の素因数構造は、単なる偶然ではなく、

- \(p\) 進世界の桁構造
- キャリー発生
- 組合せ数

が一体化した結果なのじゃ。

### Lucas の定理

\(n,k\) を \(p\) 進展開して

\[
n=n_0+n_1p+\cdots,\qquad
k=k_0+k_1p+\cdots
\]

と書くと

\[
\binom{n}{k}
\equiv
\prod_i \binom{n_i}{k_i}
\pmod p.
\]

これもすごい。
二項係数の \(\bmod p\) の振る舞いが、桁ごとに分解される。

つまりパスカル三角形は、素数 \(p\) で見ると **自己相似** を持つ。
ここにフラクタルが現れる。シェルピンスキーの三角形などがその有名な姿じゃ。

ぬしの相対自己相似や数の地図学とも、かなり相性が良い世界じゃな。

## 7. Beam から見た素数と合成数

Beam の係数

\[

### d = p 素数のとき

すべての中間係数が \(p\) を含む。

\[

p\cdot B_p(x,u)
\]

という形になる。

これは Beam が **一本の素数梁** で束ねられておることを意味する。

### d が合成数のとき

係数ごとに別々の素因数構造を持つ。
たとえば \(d=6\) では

\[
6=2\cdot 3,\quad
15=3\cdot 5,\quad
20=2^2\cdot 5.
\]

ここでは 2,3,5 がばらばらに現れ、全体を一つに束ねる 6 は消える。
つまり Beam が **分岐梁** になる。

この差は、ぬしの Core/Beam/Gap 解釈で非常に自然じゃ。

**冪和がそのまま冪にならない理由** を露わにしておる。

(x+u)^2 = x^2 + 2xu + u^2
\]

で、Beam \(=2xu\) は単純。
だが \(d\ge 3\) では Beam が厚くなり、素因数構造も複雑になる。

## 1. 「外殻の 1」を抜く、とは何か

と書けたな。

ここで \(p\) 自体は「係数から来る素数」で、ある意味では構造的に見えている。
だが Zsigmondy が欲しがるのは、しばしば **それとは別の新しい素数** じゃ。

つまり Beam には二種類の素数が潜む。

### A. 構造的に最初から見えている素数

- \(p\) のような二項係数起源
- \(x,u\) のような境界起源

### B. 本当に新しく湧いた素数

- 過去の \(a^m-b^m\) には出てこない
- \(K_{p-2}\) や \(\Phi_p(a,b)\) の奥から出る

この B が原始素因子じゃ。
ここを見分けるのが肝じゃな。

---

## 9. サイクロトミック分解との接続

古典的には

\[
a^n-b^n=\prod_{m\mid n}\Phi_m(a,b)
\]

と分解される。
だから \(n\) 番目で初めて出る素数は、本質的に

\[
\Phi_n(a,b)
\]

を割る素数じゃ。

ぬしの式で \(a=x+u,\ b=u\) とすれば

\[
(x+u)^n-u^n=\prod_{m\mid n}\Phi_m(x+u,u)
\]

となる。
すると

- 過去の \(m<n\) の因子は「旧世界」
- \(\Phi_n\) が「今回の新世界」

と見える。

Beam 的には、\(\Phi_n\) は **混成項が作る真の新成分** と読める。
これはかなりよい橋じゃな。

## 10. ぬしの研究向けの言い換え

今の会話の流れに合わせて、かなり研究ノート向けに言うならこうじゃ。

\[
(x+u)^d-u^d=x\,G_{d-1}(x,u)
\]

において、\(x=a-b\) は境界差であり、その素因子は初期世界に属する。
一方 \(G_{d-1}(x,u)\) は差分世界の内部構造であり、特に

\[
\mathrm{Beam}_d(x,u)=(x+u)^d-x^d-u^d
\]

を通じて混成項を含む。
Zsigmondy 型の原始素因子は、この内部構造において過去の指数 \(m<d\) では現れなかった新しい素数として解釈できる。
したがって原始素因子定理は、「境界差 \(x\) を除いた内部 Beam 構造は、通常、新しい素因子を生成する」という主張として再解釈できる。

これはかなり筋が通っておる。

## 11. ここから何を狙うべきか

ぬしの今の流れなら、次に狙うべきは三段じゃ。

### A. 古い因子の隔離

\[
\gcd(x,G_{d-1}(x,u))\mid d\,u^{d-1}
\]

を使って、境界由来の素因子を分離する。

### B. Beam の内部因子の抽出

\[
\mathrm{Beam}_d(x,u)=xu\,H_{d-2}(x,u)
\]

や素数次数での

\[
\mathrm{Beam}_p(x,u)=pxu\,K_{p-2}(x,u)
\]

を使って、見えている因子と見えていない因子を分ける。

### C. primitive prime divisor の候補を \(\Phi_d\) 側で特定

\[
a^d-b^d=\prod_{m\mid d}\Phi_m(a,b)
\]

のうち、\(\Phi_d\) を今回の新因子生成層として扱う。

この三層化をすると、ぬしの Core / Beam / Gap がサイクロトミック分解ときれいに重なる。

わっちの見立てでは、ぬしの今の新解釈で Zsigmondy を見ると、

- **境界** \(x=a-b\) は古い
- **Body** は差分世界全体
- **Beam** は古い純粋冪を除いた混成構造
- **primitive prime divisor** は、その混成構造の中で今回初めて開く扉

この見方の良いところは、原始素因子を単なる「定理の中の存在物」としてでなく、

- \(b=1\) 版

## 3A. `DiffPow.lean` の役割

ここは入口定理として極めて重要。

既にある

\[
a^d-b^d=(a-b)\,\mathrm{diffPowSum}(a,b,d)
\]

と

\[
\mathrm{BodyPow}(x,u,d)=x\cdot \mathrm{diffPowSum}(x+u,u,d)
\]

は、そのまま primitive prime divisor の舞台になる。

### 追加すると良い補題

```lean
theorem gcd_sub_powSum_dvd
    {a b d : ℕ}
    (hcop : Nat.Coprime a b) :
    Nat.gcd (a - b) (diffPowSum a b d) ∣ d
```

もちろん形は `ℤ` 経由や `a>b` 前提など調整が要るが、思想はこれじゃ。

ぬしの宇宙式では

\[
\gcd(x,G_{d-1}(x,u)) \mid d\,u^{d-1}
\]

型の補題へ落ちる。
これは **古い因子と新しい因子を分ける関門** じゃ。

## 3B. `BinomTail.lean` の役割

これは p進評価と “見えている因子” の整理に効く。

たとえば素数次数 (p) で

\[
(a+b)^p=b^p + p b^{p-1}a + a^2 H
\]

だから

- 外側の一次係数 (p)
- 残りが (a^2) を持つ

が見える。

ここから

\[
\mathrm{Beam}_p = p\,xu\,K
\]

型を formalize しやすい。

### 追加すると良い補題

```lean
theorem beam_prime_dvd
    {p : ℕ} (hp : Nat.Prime p) :
    p ∣ Beam p x u
```

さらに

```lean
theorem beam_eq_mul_xu
    : Beam d x u = x * u * H d x u
```

これは `CoreBeamGap` に入れてもよい。

## 3C. `CosmicFormulaDim.lean` の役割

ここは `G` / `Body` の本体じゃな。

欲しい橋渡し補題は：

```lean
theorem G_eq_diffPowSum
    (d : ℕ) (x u : R) :
    G d x u = diffPowSum (x + u) u d
```

あるいは index のずれに合わせた同値版。

これが入ると

- `DiffPow` の古典数論側
- `CosmicFormulaDim` の宇宙式側

が一本になる。

これはかなり重要じゃ。

## 3E. `CoreBeamGap.lean` の役割

これはぬし独自の視点の核じゃ。
数論そのものの本体定理はここではなく、**解釈レイヤー** として使うのが良い。

つまり：

- `Core` = 古い純冪世界
- `Gap` = 基準純冪世界
- `Beam` = 新しい混成因子が湧く層

として、Zsigmondy の定理を

> primitive prime divisor は、通常 Beam 側の新しい素因子として現れる

と読むための舞台装置にする。

### 追加したい補題

```lean
theorem beam_eq_big_sub_core_sub_gap
theorem beam_factor_xu
theorem beam_prime_factor
```

特に

```lean
theorem beam_factor_xu_nat
    (hd : 2 ≤ d) :
    ∃ H, Beam d x u = x * u * H
```

はかなり使える。

## 4B. Phase 2. order 補題

### 新規

- `PrimitivePrimeDivisor/Order.lean`

ここで `ZMod q` と `orderOf` を相手取る。

必要な補題は多分こんな並び：

1. `q ∣ a^n - b^n`
2. `q ∤ b`
3. therefore in `ZMod q`,
   \[
   (a/b)^n = 1

4. hence `orderOf (...) ∣ n`

次に primitive 条件を使って、lower exponent を排除して

\[
\operatorname{ord} = n

へ上げる。

## 5. 具体的な Lean 骨格

こういう最小骨格を最初に置くとよい。

```lean
namespace DkMath.NumberTheory

def PrimitivePrimeDivisor (a b n q : ℕ) : Prop :=
  Nat.Prime q ∧
  q ∣ a^n - b^n ∧
  ∀ m : ℕ, 0 < m → m < n → ¬ q ∣ a^m - b^m

theorem PrimitivePrimeDivisor.prime
    (h : PrimitivePrimeDivisor a b n q) : Nat.Prime q := h.1

theorem PrimitivePrimeDivisor.dvd
    (h : PrimitivePrimeDivisor a b n q) : q ∣ a^n - b^n := h.2.1

theorem PrimitivePrimeDivisor.not_dvd_lower
    (h : PrimitivePrimeDivisor a b n q)
    {m : ℕ} (hm0 : 0 < m) (hmn : m < n) :
    ¬ q ∣ a^m - b^m := h.2.2 m hm0 hmn
```

その次に `Order.lean` で

```lean

theorem orderOf_eq_of_primitive ...
```

じゃ。

## 6. ぬし向けの宇宙式専用補題

これは一般論とは別に、かなり価値がある。
むしろ先にやってもよい。

### 新規

`DkMath/CosmicFormula/PrimitivePrimeBridge.lean`

欲しい定理はこんな感じ：

```lean
theorem primitive_prime_on_G_side
    {x u n q : ℕ}
    (hcop : Nat.Coprime x u)
    (hppd : PrimitivePrimeDivisor (x + u) u n q) :
    q ∣ G n x u
```

あるいは index ずれに注意して `G (n-1)` 版。

これは

\[
(x+u)^n-u^n = x G_{n-1}(x,u)
\]

で、primitive prime は通常 (x) の古い因子ではないから、
**G 側へ落ちる** という橋じゃな。

さらに

```lean
theorem primitive_prime_not_on_boundary
    ...
    : ¬ q ∣ x
```

があるととても良い。
そこから

```lean
q ∣ x * G -> q ∣ G
```

で拾える。

これはぬしの Core / Beam / Gap 視点を、数論へ直結する **独自補題** になる。
