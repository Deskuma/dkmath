# review-014: ABCCoreInsertionCandidates.md

うむ、これは **良い洗い出し** じゃ。
しかも今回は、ただ「次はここが怪しい」と勘で挙げたのではなく、`ABC016.lean`, `ABC002.lean`, `ABC038.lean`, `RatioBound.lean` を点検したうえで、差し込み候補を 3 件に絞り、各候補について「今の bridge API だけで足りるか」「何がまだ足りないか」を明記しておる。整理の粒度としてかなり良い。

結論から言うと、わっちもこの優先順位に **概ね賛成** じゃ。
特に `RatioBound.count_with_rad_eq_le_div` を最優先に置いた判断は堅い。理由は単純で、今の bridge spine が直接供給するのは `ABC.rad(diff)` の **下界** であり、`RatioBound` 側はまさに `rad` の大きさを count bound に変換する層だからじゃ。型の近さという意味で、今回の 3 候補の中ではここが最も実装に近い、という診断は妥当じゃ。

`ABC038.quality_le_of_not_bad_with_tail` を「本命候補だが、差し込み直前には family がどの triple / diff に付くかの整理が要る」としたのも、見立てとして正しい。
quality 系は最終的に `rad(...)` を介して動くので bridge が効く余地は大きいが、現状の bridge が出すのは `ABC.rad (a^d-b^d)` であり、候補側が欲しているのは `ABC.rad (a*b)` 系じゃ。このズレを埋める中間命題なしに drop-in はできぬ、という判断は冷静でよい。むしろ、ここを無理に「本命だから先に行ける」と言わなかったのが賢い。

`ABC016.twoTail_le_rad_pow_of_log_bound` の位置づけも納得できる。
ここは構造的には面白い。`twoTail` や square-excess budget の読み替えに `rad` 下界が効く可能性はある。じゃが現状では、対象 quantity のズレに加えて、向こうが上界制御でこちらが下界制御じゃから、噛み合わせにはもう 1 本中間命題が要る。候補としては重要だが、最短手ではない、という評価はその通りじゃ。

つまり今回の整理は、三候補をただ並べたのではなく、

* `RatioBound` は **いまの API に近い**
* `ABC038` は **quality 本体への入口だが 1 本橋が足りぬ**
* `ABC016` は **構造的に重要だがズレが最も大きい**

というふうに、ちゃんと「着手順」に変換できておる。ここがよい。

今回のレビューで、わっちが特に評価する点は二つある。
ひとつは、`ABC002` まで点検対象に入れつつも、最終候補からは外しておることじゃ。これは「見たが、今の bridge spine の直近の差し込み先ではない」と判断できたということじゃろう。洗った範囲と、実際の優先順位付けが分離できておるのは良い。
もうひとつは、今回 Lean コードを無理に足しておらぬことじゃ。いま必要なのは theorem を増やすことではなく、**どの補題が足りないかを間違えずに特定すること** なので、調査と文書整理だけで止めたのは正しい。

留意点もある。
`RatioBound` を最優先にしたのは良いが、まだ「かなり近い」であって「そのまま刺さる」ではない。差分文書でも書いておる通り、必要なのは

$$
\operatorname{rad}(a) \ge R
\Rightarrow
\text{その count class は } X/R \text{ 型で縮む}
$$

という薄い corollary じゃ。
つまり次の一手は、`PrimitiveWitnessFamily` から直接 `count_with_rad_eq_le_div` を呼ぶことではなく、その前に **`rad lower bound -> count upper bound` の一般 corollary** を 1 本作ることじゃな。ここを飛ばすと、結局また型が噛み合わぬ。

わっちの最終評価はこうじゃ。
**この洗い出しは成功しておる。**
とくに「最も実装に近いのは `RatioBound` 周辺」「quality 本体はもう 1 本中間命題が必要」と明確に切れたのが大きい。これで次は、迷わず `RatioBound` 側に薄い corollary を足しに行ける。

次の Codex 指示としては、これがよい。
`DkMath.ABC.RatioBound` に、`ABC.rad a ≥ R` を仮定したとき `a ≤ X` の count class が `X / R + 1` 型で抑えられる薄い corollary を追加する。証明は既存 `count_with_rad_eq_le_div` を使い、`rad a = r` ごとの分割をせずに済む最小形を探る。もし一般形が重ければ、まずは `rad a ≥ R` かつ `a ≤ X` を満たす集合に対する粗い単調性版でもよい。`

ひとことで言えば、
**今回は橋を刺す場所の地図ができた。次は、その地図でいちばん近い城門 `RatioBound` を実際に叩く番じゃ。**
