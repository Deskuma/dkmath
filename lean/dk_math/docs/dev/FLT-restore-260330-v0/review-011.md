# GTail Phase C を終えて

うむ、状況は **かなり良い前進** じゃ。
しかも今回は、ただ名前を切り分けただけではなく、**GTail 側に本物の数学的な足場** が増えた。`GTail.lean` に合同性補題群と mod (p^2) / mod (p^3) 展開が入り、Phase C は完了、全体ビルドも成功、sorry 追加も無い。これは「橋を先に架ける」という方針にちゃんと沿った前進じゃよ。

## 1. 何が増えたか

今回の核は 3 本ある。

まず、`GTail_congr_of_modEq` とその周辺で、**GTail は (x,u) の合同を素直に伝播する多項式** だ、という地盤が Lean 上で固まった。
次に、`n \mid x` のとき

$$
GTail, d,1,x,u \equiv \binom{d}{1}u^{d-1} \pmod n
$$

へ崩壊する基本定理群が整った。
そして最後に、prime (p) で (p \mid x) のとき

$$
GN \equiv p,u^{p-1} \pmod{p^2}
$$

さらに条件付きで mod (p^3) までの頭項展開が入った。
これは Branch A の Wieferich 解析に直接つながる、かなり強い補題群じゃ。

## 2. なぜ重要か

前までは、

$$
GN = p,y^{p-1} + \text{高次補正}
$$

と口では言えても、それを Lean で扱うための **合同性の骨組み** がまだ薄かった。
今回それが埋まったことで、GN を

* 円分核の顔
* 差分商の顔
* mod (p^k) 展開の顔

で行き来するための、実務的な橋が太くなったのじゃ。
とくに `GN_eq_head_add_p_sq_mul_of_prime_dvd_x` と `GN_eq_head_add_p_cube_mul_of_dvd_tail` のような **explicit な (M) 付き等式版** があるのは大きい。これで「合同」から「実際の形」へ持ち上げて議論できる。

## 3. いまの位置づけ

これは本丸を落としたわけではない。
じゃが、**本丸へ向かう前の地形図** はだいぶ整った。

いま Branch A 側で本当に重いのは、依然として contradiction kernel じゃ。
ただ、その矛盾探索をやるにも、まず

$$
GN \bmod p^2,\quad GN \bmod p^3
$$

を正確に持っておらねば話にならぬ。
今回の実装は、まさにその準備を完了した段じゃ。
ゆえにこれは「回り道」ではなく、**本丸の前に必要な橋桁工事** と読むのが正しい。

## 4. 技術的にも良い兆候

地味に良いのは、Mathlib にそのまま使える `Finset.sum` と `Nat.ModEq` の橋が無かったため、`sum_range_modEq` を private helper として手実装し、その上で GTail の合同伝播を立てた点じゃ。
これは単に補題が増えたというより、**今後の合同性解析の下請け基盤** ができたということでもある。

## 5. 次に何をすべきか

次の課題は、かなりはっきりしておる。

第一に、計画書どおり Phase A へ戻って、

$$
\text{円分核} \leftrightarrow GN \leftrightarrow \text{差分商}
$$

の橋を theorem 化すること。
第二に、今回できた `GN_mod_p2_head` などを Branch A の Wieferich 条件

$$
u^{p-1} \equiv 1 \pmod{p^2}
$$

と接続する具体補題、たとえば `branchA_spow_congr_head_mod_p2` 型を起こすこと。
この 2 つが揃うと、GTail 高次解析が「ただの補題集」ではなく、contradiction 探索へ直接刺さる武器になる。

## 6. 総括

ひとことで言えば、こうじゃ。

**今回は本丸を倒したのではなく、本丸へ届く橋を 1 本ちゃんと架けた。しかもその橋は、GN の高次展開と合同性という、後戻りしにくい良い橋じゃ。** GTail Phase C が終わった今、次は Phase A の橋渡し定理と、Wieferich 条件との接続補題へ進むのが最も筋がよい。

よい流れじゃよ。
いまは「殴る」より「繋げる」が勝つ局面、その読みは当たっておる。
