# Discussion: No.090 cp

## 解像度を上げると曲がって見える

## うむ、そういうことじゃ

ぬしの言い方はかなり核心に近い。

`mod 2^n` で観測するというのは、普通の意味で「遠くまで見る」というより、

```text
同じ整数世界を、2-adic な解像度で細かく見る
```

ということじゃ。

つまり、

```text
mod 4
mod 8
mod 16
mod 32
...
```

と上げていくのは、観測範囲を横に広げているというより、**同じ点の住所をより細かく読む** 操作に近い。

## 「範囲が広がる」のではなく「住所が細かくなる」

自然数の通常感覚では、

```text
1, 2, 3, 4, 5, ...
```

と横に進む。

しかし `mod 2^n` では、

```text
mod 4 で見る
  -> 1 or 3

mod 8 で見る
  -> 1, 3, 5, 7

mod 16 で見る
  -> 1, 3, 5, 7, 9, 11, 13, 15
```

というように、**各 residue cell がさらに分岐する**。

だからこれは、直線を遠くまで見るというより、

```text
一本の道だと思っていたものを拡大したら、
中に分岐構造が見えた
```

という感じじゃ。

## 曲がっているのは何か

問いはまさにそこじゃな。

```text
観測窓の上がり方が曲がっているのか？
解像度の上がり方が曲がっているのか？
数の成長が曲がっているのか？
```

賢狼の現時点の答えはこうじゃ。

```text
自然数の成長は、通常座標では真っ直ぐに見える。
しかし 2-adic 座標で見ると、真っ直ぐではなく分岐木として見える。
そのズレが「曲がり」として現れている。
```

つまり、数そのものが物理的に曲がっているというより、

```text
自然数の通常距離
2-adic な解像度
Collatz の 3n + 1
2^h による剥離
```

この四つのスケールゲージが一致していない。

その不一致が「曲がり」として観測される。

## Collatz の曲がり

Collatz では、奇数 (n) に対して、

```text
3n + 1
```

を作り、そこから (2) を何個剥がすかを見る。

このとき、

```text
3 による膨張
2-adic residue による剥離
```

が噛み合っていない。

ここが曲がりの源じゃ。

たとえば `mod 8` では、同じ `height = 1` でも、

```text
3 mod 8:
  今は height = 1
  次に height >= 2 を生む

7 mod 8:
  今は height = 1
  次も height = 1 側に残る
```

に分かれた。

つまり、低解像度では同じに見えたものが、高解像度では違う未来を持つ。

これが分岐。

## 観測範囲を広げると結果が分岐する

うむ。まさにそれじゃ。

ただし、ここでいう「観測範囲」は普通の距離ではなく、

```text
2-adic 解像度の深さ
```

じゃ。

`mod 4` では二つの cell しか見えない。

```text
1 mod 4:
  height >= 2

3 mod 4:
  height = 1
```

`mod 8` では四つに分かれる。

```text
1 mod 8:
  height = 2

3 mod 8:
  delayed-peeling

5 mod 8:
  height >= 3

7 mod 8:
  retention
```

さらに `mod 16` では、`7 mod 8` の内部が分かれる。

```text
7 mod 16:
  retention から recovery 側へ向かう

15 mod 16:
  retention continuation 側へ向かう
```

こうして、観測解像度を上げるほど、未来の分岐が見える。

## これは「座標系の曲率」に近い

直感的に言えば、今見ているのは

```text
自然数直線を 2-adic レンズで見たときの座標系の曲率
```

じゃと思う。

通常の自然数軸では、数はただ増える。

しかし `mod 2^n` の世界では、数は

```text
どの residue cylinder に入るか
```

で分類される。

そこに Collatz 写像 (3n+1) が入ると、各 cylinder が次の cylinder へ送られる。

つまり、

```text
自然数の直線的成長
  -> 2-adic residue tree では分岐遷移として見える
```

この変換が、まさに「曲がり」じゃ。

## 成長が曲がっているのか、観測が曲がっているのか

ここは分けて言える。

自然数の通常加算・乗算だけを見るなら、数の成長は真っ直ぐに見える。

しかし Collatz の加速写像は、

```text
3n + 1 を作る
その後に 2^h で割る
```

という、通常距離と 2-adic 距離を混ぜる操作じゃ。

だから、

```text
観測座標も曲がっている
写像もその座標に対して斜めに走っている
```

というのが正確だと思う。

座標系だけの問題でもない。
数の成長だけの問題でもない。

```text
3 による Archimedean な膨張
2 による 2-adic な剥離
```

この二つのゲージが重なったときに、軌道が曲がって見える。

## コラッツ予想が教えていること

ここから見ると、コラッツ予想はこう言っているように見える。

```text
どれだけ retention 側へ逃げても、
より高い 2-adic 解像度で見れば、
その逃げ道はさらに分岐し、
多くの枝は delayed-peeling や immediate-peeling へ戻される。
```

つまり、

```text
低剥離 path は存在する。
しかし、それを続けるには、どんどん細い residue cylinder に入らねばならない。
```

この「どんどん細くなる逃げ道」を Lean で形式化できれば、かなり本丸に近づく。

## DkMath 的に言うと

いま見えている構造はこうじゃ。

```text
Core:
  odd orbit label

Beam:
  3n + 1 による膨張

Gap:
  mod 2^r の解像度で見える residue 分岐

Tail:
  遅れて現れる剥離

Big:
  sumS による累積 drift 回収
```

そして Petal は、

```text
mod 2^r の解像度を上げながら、
どの branch がどの branch へ遷移するかを見る座標系
```

になっている。

## まとめ

ぬしの言葉で言えば、こうじゃ。

```text
mod 2^n で観測窓を広げるとは、
自然数軌道を 2-adic 解像度で拡大すること。

解像度を上げるほど、
同じに見えた挙動が分岐する。

その分岐は、
数の成長と観測ゲージのズレから生まれる。

Collatz は、
そのズレによって生じる曲がった遷移構造を通じて、
不足した縮小力を遅延回収している。
```

うむ。
かなり本質に近い。

次に Lean でやるべきことは、この直感を

```text
retention が続くほど、
必要 residue が mod 2^r の細い cylinder に押し込まれる
```

という形で補題化していくことじゃ。

## 素数階乗と素数剰余類の分類構造との比較

これと似たようなのが、素数の剰余類だね。

p mod Primorial

素数を素数階乗で分類するとほぼ平均的に一様に分かれていく。
分類解像度を上げて仕分けていく。こちらは偏りが少ない。

最終的には全ての素数がそれぞれ、己が為々（しし）の剰余類へと収まり、
各素数の剰余類にはその素数が１個となっていく。

このような現象と対比されるような構造に見える。

## うむ、かなり似ている

まさに似た構造じゃ。
ただし、**素数の剰余類分類**と**Collatz の \(2^n\) 剰余類分類**では、分岐の性質が違う。

大きく言うと、

```text id="bsu7zh"
素数 mod Primorial:
  解像度を上げると、ほぼ一様に散っていく

Collatz mod 2^n:
  解像度を上げると、遷移の役割が分岐していく
```

という対比じゃな。

## 素数側：Primorial は「ふるいの座標系」

素数を

```text id="thj6b3"
p mod Primorial
```

で見るとき、たとえば

```text id="mae3v1"
M = 2 * 3 * 5 * 7 * ...
```

という素数階乗、つまり primorial を法にする。

すると、\(M\) に含まれる小さい素数で割れる剰余類は、素数候補から消える。

残るのは、

```text id="ze9hbp"
gcd(a, M) = 1
```

を満たす剰余類。

ここに、十分大きな素数たちが入っていく。

これはまさに **ふるいの座標系**じゃ。

```text id="4s8q64"
解像度を上げる
  -> 小さい素因子による禁制クラスが増える
  -> 生き残りの residue class が細分化される
```

## 素数側は「平均的に一様」

ここが Collatz と大きく違う。

固定した modulus \(M\) に対して、素数は互いに素な剰余類へ、漸近的にはかなり均等に分布する。

つまり、

```text id="0t61yb"
a mod M
b mod M
c mod M
...
```

のうち、\(\gcd(a,M)=1\) であるクラスには、素数が長期的にはおおむね一様に現れる。

もちろん有限範囲では偏りがある。
しかし構造の主成分は、

```text id="x4zz6u"
各 admissible residue class に素数が散っていく
```

という方向じゃ。

これは Collatz のように、

```text id="rr8j62"
この class は次に縮む
この class は retention する
この class は delayed-peeling する
```

という **動的な役割分担**とは少し違う。

## Collatz 側：mod \(2^n\) は「遷移の座標系」

Collatz の \(\mod 2^n\) は、単なる分類ではなく、次の挙動を決める。

たとえば mod 8 では、

```text id="deyfou"
1 mod 8:
  exact height 2

3 mod 8:
  delayed-peeling

5 mod 8:
  immediate strong peeling

7 mod 8:
  retention
```

になる。

つまり、Collatz の剰余類は単に「どこにいるか」ではなく、

```text id="cs8sui"
次にどちらへ曲がるか
どのくらい 2 を剥がすか
縮小力を今出すか、次に遅延して出すか
```

を持っている。

こちらは **動的座標系**じゃ。

## 対比するとこう

```text id="o6stvw"
素数 mod Primorial:
  静的なふるい座標
  禁制クラスを除く
  残ったクラスへほぼ一様に散る
  分類解像度を上げると細かく均等化する

Collatz mod 2^n:
  動的な遷移座標
  各クラスが次状態を持つ
  delayed / retention / immediate peeling に分かれる
  分類解像度を上げると役割が分岐する
```

これじゃな。

## 「己が為々の剰余類」について

ぬしの言う、

```text id="wakxqt"
最終的には全ての素数がそれぞれ、己が為々の剰余類へ収まる
```

という表現は、有限観測としてはとてもよく分かる。

たとえば、観測範囲を \(p \le X\) に固定して、primorial \(M\) を \(X\) より大きくすれば、各素数 \(p\) はそのまま

```text id="vz1gwa"
p mod M = p
```

として別々の剰余類に入る。

つまり、十分高い解像度では、

```text id="b9q2cq"
各素数が個別住所を持つ
```

ように見える。

ただし無限全体では、最終的な一つの有限 modulus があるわけではない。
解像度を上げ続けることで、有限範囲の素数がどんどん個別化される、という構図じゃ。

## Collatz との本質的な違い

素数の場合、解像度を上げると、

```text id="w5h9oh"
候補がより細かく分類される
```

しかし各クラスは、おおむね対等な「住所」になる。

Collatz の場合、解像度を上げると、

```text id="ve3iv9"
同じに見えていた住所の中に、未来の違いが現れる
```

たとえば、

```text id="r5zxve"
3 mod 4:
  height = 1
```

だけでは粗い。

mod 8 に上げると、

```text id="bhl9m5"
3 mod 8:
  次に縮む

7 mod 8:
  次も低剥離に残る
```

と分かれる。

さらに mod 16 に上げると、`7 mod 8` の内部がまた分かれる。

```text id="ezbqdm"
7 mod 16:
  recovery 側

15 mod 16:
  retention continuation 側
```

これは、素数の「均等に散る分類」と違って、

```text id="b1ja5c"
悪い経路がより細い cylinder へ押し込まれる分類
```

じゃ。

## かなり良い対比

だから、対比としてはこう言える。

```text id="tsvq7d"
素数:
  Primorial 解像度を上げると、
  生き残る剰余類が広く均等に開いていく。

Collatz:
  2^n 解像度を上げると、
  軌道の未来が分岐し、
  低剥離を続ける道が狭い枝へ押し込まれていく。
```

素数は **拡散**。
Collatz は **分岐と圧縮**。

そんな感じじゃ。

## DkMath 的に言うなら

素数側は、

```text id="k4vr7w"
Primorial address:
  divisibility obstruction を取り除いた残りの住所空間
```

Collatz 側は、

```text id="gp5dk1"
2-adic Petal address:
  transition obstruction と delayed drift を読む住所空間
```

素数では、住所空間が広がるほど「候補が均等に散る」。
Collatz では、住所空間が深くなるほど「未来の役割が細分化される」。

## まとめ

ぬしの見立てはかなり鋭い。

```text id="q8tbcl"
p mod Primorial は、素数をふるい解像度で仕分ける座標系。

n mod 2^r は、Collatz 軌道を 2-adic 遷移解像度で仕分ける座標系。
```

そして対比は、

```text id="6ozzfg"
素数:
  解像度を上げるほど、均等に散る。

Collatz:
  解像度を上げるほど、未来の枝が分かれる。
```

さらに踏み込むと、

```text id="z1pya1"
素数は、分類解像度の上昇によって個別性へ向かう。

Collatz は、分類解像度の上昇によって運命分岐へ向かう。
```

うむ。
これはかなり良い比較軸じゃ。
DkMath の Petal 観測は、素数側では Primorial sieve address、Collatz 側では \(2^n\)-adic transition address として並べられる。
